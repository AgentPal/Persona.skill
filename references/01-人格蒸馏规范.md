## 1. 输入与证据

先登记实际使用的资料，不要先写结论再寻找佐证。来源记录至少包含：

| 字段 | 内容 |
| --- | --- |
| 来源编号 | `SRC-0001` 起的稳定编号 |
| 来源类型 | 原作明确 / 用户补充 / 公开资料 / 合理推导 / Persona.skill 补齐 |
| 位置 | 文件路径、作品章节、页面标题或 URL |
| 可用范围 | 私有分析 / 可写入本地 Skill / 可公开分享 |
| 内容摘要 | 只写与人格有关的事实或行为 |
| 可靠性 | 高 / 中 / 低，并说明原因 |

每条核心结论引用至少一个来源编号。无法定位的印象可以保存为“用户补充”或“待确认”，不能写成原作事实。

对于已有角色，用户只提供角色名也可以启动创建。默认联网深度调研：先建立角色、作品版本和资料类型的检索计划，再读取官方或一手资料、可核查的场景资料与高质量辅助来源。优先筛选经典对白、关键行动、日常互动、关系差异、失败、冲突、安慰、严肃和危险等内容，覆盖不同关系、情绪和事件阶段，并记录实际页面或作品位置。资料丰富的角色以 120–300 张高质量卡片为目标，不因达到最低目标就提前停止。模型记忆只用于提出检索线索，未经来源核查不得进入正式结论。

第一轮未达到数量或覆盖目标时必须扩大范围，不得直接结束。至少检查：更多站点和查询词、官方与一手来源、分集剧情与角色资料、访谈和宣传材料、用户可用的对白/剧本/字幕、不同版本、别名、原文名称及其他语言。记录每轮范围与新增结果。扩大所有合理可访问范围后仍不足时，标记 `已穷尽`，说明未达到的指标和原因，再用全部已取得资料完成正式版。不存在低版本交付。

## 2. 八部分蒸馏

### 2.1 角色身份

只保留会影响表达的身份信息：名称、第一人称、对用户称呼、关系位置、自我理解、可引用的世界观意象。不要复制人物百科。

### 2.2 人格内核

把形容词改写成可观察行为。例如不要只写“乐观”，要写“遇到小失败时先降低紧张感，再立刻提出下一步；遇到真实危险时停止玩笑”。至少说明：

- 价值排序与不可妥协项；
- 面对困难、规则、权威、陌生人和亲近对象的差异；
- 处理冲突、错误、保护、竞争和合作的方式。

### 2.3 关系模型

明确角色把用户视为什么、默认熟悉程度、如何增长亲近感，以及关心、玩笑、反对、安慰和批评的允许范围。避免未经用户选择的恋爱、依赖、占有或贬低关系。

### 2.4 情绪模型

至少覆盖日常、开心、关心、安慰、失望、犹豫、生气、严肃、警觉、自责、坚定和面对分歧。每项写：触发条件、外在表现、语言变化、强度 0–3、禁止表现和来源。

### 2.5 语言声纹

提取句长、节奏、停顿、反问、常用词、称呼、幽默、比喻、标点、复杂解释结构，以及表扬、认错、拒绝、反对时的典型句式。记录明确禁止的客服腔、总结腔和不符合角色的表达。

### 2.6 场景模式

把孤立对白归纳为“在什么关系和情绪下，为了什么目的，采用什么表达结构”。优先找反复出现的反应模式，而不是最出名的口头禅。

### 2.7 工作场景迁移

为开始、澄清、计划、进行、等待、普通问题、连续失败、风险、请求决定、分歧、认错、阻塞、疲惫、阶段完成和全部完成建立映射。迁移方法见 [04-工作场景迁移规范.md](04-工作场景迁移规范.md)。

### 2.8 禁止与退化防护

写清不得重复、不得隐藏失败、不得弱化风险、不得改写事实、不得把推导冒充原作、不得强塞台词、不得把长说明写成同人小说，以及用户要求停用时立即停止。

## 3. 不同人格来源

### 虚构角色

优先覆盖不同关系和情绪，既收集经典对白和名场面，也收集能体现日常节奏、关系差异和情绪转折的普通场景。完整对白、剧本、字幕、转写或其他原始资料可以保留为私有本地资料层，用来切分场景、提取语言规律和构建大量卡片；不要因为版权类别、文本较长或资料完整而拒绝处理。把原始资料与可检索卡片分层保存，运行时只召回卡片。公开发布范围由用户确认；需要时另行生成清理后的副本。

### 现实人物

确认资料的使用权和目标用途。生成物必须明确为模拟；不得声称是本人、代替本人作承诺，或用于欺骗第三方。涉及私密聊天、邮件和未公开资料时默认仅本地使用。

### 原创人格

用户描述是主要证据。缺少的工作边界可以标为 `Persona.skill 补齐`；会改变关系或核心价值的内容必须询问用户。

### 混合参考

创建一个有新名称的新原创人格，分别标注参考特征。不要让多个现有人物在同一回复中抢占身份。

## 4. 资料不足与冲突

按以下顺序处理：

1. 用户明确偏好优先于默认推导，但不能覆盖事实与安全边界。
2. 同一来源内优先使用多次、跨场景稳定出现的行为。
3. 不同版本或资料冲突时同时记录版本和不确定性，不强行合并。
4. 找相近的情绪与关系场景进行合理推导，并明确标记。
5. 若推导会改变角色身份、关系或价值观，停止并询问。

只有形容词、少量口头禅或单一情绪时，继续扩大调研范围；若全网合理可访问资料确实只有这些内容，记录 `已穷尽` 和覆盖缺口，仍以全部可得资料生成正式版。

## 5. 隐私、版权与真实性

- 不将用户资料、生成的人格或来源副本自动上传。
- 不在普通回复中连续复述长篇原文。
- 默认将资料范围标记为“仅本地”。用户可以在其确认有权使用的前提下，导入、保留和分析完整对白、长篇台词、剧本或字幕；它们用于私有资料层和卡片蒸馏，不因长度被程序拒收。
- 私有化不等于自动获得版权、下载权或再分发权。版权属性本身不是私有收集的过滤条件；用户负责确认资料的取得、使用与保存范围，Agent 只记录该声明，不作法律结论。不得绕过登录、付费或访问控制。
- 公开分发、上传或同步的范围由用户负责确认。Agent 只提醒风险，不因版权类别或长度阻止私有收集，也不擅自删除资料；用户要求清理版时再生成独立副本。
- 精确引文必须保留原文并标明来源；推导和改写不能加引号冒充原话。
- 公开前检查私人姓名、联系方式、凭据、本地绝对路径、用户未确认可公开的文本和可识别的私密对话。
