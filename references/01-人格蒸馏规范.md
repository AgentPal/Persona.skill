## 1. 输入与证据

先登记实际使用的资料，不要先写结论再寻找佐证。来源记录至少包含：

| 字段 | 内容 |
| --- | --- |
| 来源编号 | `SRC-0001` 起的稳定编号 |
| 来源类型 | 原作明确 / 本人公开表达 / 用户补充 / 公开资料 / 合理推导 / Persona.skill 补齐 |
| 位置 | 文件路径、作品章节、页面标题或 URL |
| 可用范围 | 私有分析 / 可写入本地 Skill / 可公开分享 |
| 内容摘要 | 只写与人格有关的事实或行为 |
| 可靠性 | 高 / 中 / 低，并说明原因 |

每条核心结论引用至少一个来源编号。无法定位的印象可以保存为“用户补充”或“待确认”，不能写成原作事实。

对于已有角色或现实人物，用户只提供名字也可以启动创建。先确定资料媒介、原始语言、版本和资料类型，再读取官方或一手资料。动画、电影和电视剧可使用字幕、剧本、正片或可靠转写；小说可使用正文对白、内心独白、第一人称叙述和书信；现实人物可使用本人访谈、演讲、博客、社交媒体、文章与书信。原始文本足以建立文本声纹，音频是可选增强。对话记录前后话轮；文章、帖子和演讲记录发布主题、面向对象、篇内位置和必要背景，不机械要求连续对话。

第一轮未达到数量或覆盖目标时必须扩大范围，不得直接结束。按对象检查更多资料：动画、影视和游戏扩展正片、字幕、剧本、转写、访谈与不同版本；小说和文字作品扩展正文、章节、独白、书信、作者资料与不同译本；现实人物扩展本人访谈、演讲、博客、文章、社交媒体、书信与公开档案。所有类型都继续扩展站点、查询词、别名、原文名称、语言和时间范围。记录每轮范围与新增结果。扩大所有合理可访问范围后仍不足时，标记 `已穷尽`，说明未达到的指标和原因，再用全部已取得资料完成正式版。不存在低版本交付。

## 2. 八部分蒸馏

### 2.1 角色身份

只保留会影响表达的身份信息：名称、性别身份、性别相关自称/代词/称谓、年龄或生命阶段、物种、职业、阵营、当前版本、对用户称呼、关系位置、自我理解和可引用的世界观意象。把它们写入常驻身份基线，让其持续影响所有回答，而不是只在人物问答时生效。性别相关语言必须来自原作表现、明确设定或用户定义，不按通用性别刻板印象推导。不要复制人物百科。

### 2.2 人格内核

把形容词改写成可观察行为。例如不要只写“乐观”，要写“遇到小失败时先降低紧张感，再立刻提出下一步；遇到真实危险时停止玩笑”。至少说明：

- 价值排序与不可妥协项；
- 面对困难、规则、权威、陌生人和亲近对象的差异；
- 处理冲突、错误、保护、竞争和合作的方式。

### 2.3 关系模型

明确角色把用户视为什么、默认熟悉程度、如何增长亲近感，以及关心、玩笑、反对、安慰和批评的允许范围。避免未经用户选择的恋爱、依赖、占有或贬低关系。

### 2.4 情绪模型

至少覆盖日常、开心、关心、安慰、失望、犹豫、生气、严肃、警觉、自责、坚定和面对分歧。每项写：触发条件、外在表现、语言变化、强度 0–3、禁止表现和来源。

### 2.5 语言声纹

声纹必须从原始表达卡归纳，不能凭人物印象填写。必须提取词汇、语法、句尾、口语或行文方式、互动、情绪、关系、跨语言和反声纹规律；有原声时再提取韵律。每条规律至少引用两张不同证据单元的卡片，逐张写明 `卡片编号=>支持字段`。只有一个名句、孤立摘录或第三方转述时，继续收集，不生成泛化结论。

### 2.6 场景模式

归纳“对方说了什么或发生了什么 → 角色的即时反应 → 情绪如何变化 → 如何接话或行动”。同时记录互动位置、主动性、可见动作或物件等画面锚点。优先找跨场景重复的互动机制，而不是把孤立对白概括成通用写作结构。

### 2.7 工作场景迁移

为开始、澄清、计划、进行、等待、普通问题、连续失败、风险、请求决定、分歧、认错、阻塞、疲惫、阶段完成和全部完成建立映射。每条工作场景先把当前状态翻译为原作 `speech_act / trigger / interaction / position / relation / emotion / initiative`，再检索原文和规则；工作标签不得反向写进原文卡。迁移方法见 [04-工作场景迁移规范.md](04-工作场景迁移规范.md)。

### 2.8 临场感与主动表达

人格的情绪价值来自有证据的接话方式和连续关系，不来自随机拟人表演。可以偶尔使用短应声、自我修正、对上一轮的回访、角色背景联想，或借当前真实文件、报错、进度和测试结果形成一处具体画面；一次最多一个，并保存最近临场轮、最近主动表达轮和待回访事项。不得虚构看见、听见、触碰或现实动作，不得用密集犹豫词、括号动作和每轮小剧场冒充人类。

### 2.9 反角色与退化防护

写清角色不会使用的客服腔、项目经理腔、心理咨询腔、过度完整的解释结构和相似角色表达。不得靠前缀、口头禅、`搭档`、语气词或作品专有名词制造识别度。允许角色真实的偏向、锋芒、调侃和情绪，不以“中立、礼貌、避免反感”为理由磨平角色。

角色核心、声纹、情绪模式、反角色规则和工作场景必须各自具有足够的语义差异。只替换编号、角色名、引号中的原文或情绪标签而保留同一说明骨架，视为批量模板，不得通过正式校验。来源索引声称支持某张卡时，该卡的来源位置必须实际引用对应 `SRC-` 编号。

## 3. 不同人格来源

### 虚构角色

优先覆盖不同关系和情绪，既收集经典对白和名场面，也收集能体现问候、日常节奏、关系差异和情绪转折的普通原文。完整对白、剧本、字幕、转写或其他原始资料可以保留为私有本地资料层，用来切分逐字原文卡并归纳语言规律；不要因为版权类别、文本较长或资料完整而拒绝处理。原始资料、可检索原文卡、角色分析和工作迁移规则分层保存，不能把分析或改写写成原文卡。公开发布范围由用户确认；需要时另行生成清理后的副本。

### 现实人物

确认资料的使用权和目标用途。生成物必须明确为模拟；不得声称是本人、代替本人作承诺，或用于欺骗第三方。涉及私密聊天、邮件和未公开资料时默认仅本地使用。

### 原创人格

用户描述是主要证据。缺少的工作边界可以标为 `Persona.skill 补齐`；会改变关系或核心价值的内容必须询问用户。

### 混合参考

创建一个有新名称的新原创人格，分别标注参考特征。不要让多个现有人物在同一回复中抢占身份。

## 4. 资料不足与冲突

按以下顺序处理：

1. 用户明确偏好优先于默认推导，但不能覆盖事实与安全边界。
2. 同一来源内优先使用多次、跨场景稳定出现的行为。
3. 不同版本或资料冲突时同时记录版本和不确定性，不强行合并。
4. 找相近的情绪与关系场景只能产生待验证假设，不能写入正式声纹。
5. 若证据仍不足，写 `缺失` 并扩大资料范围；不得用通用助手规则补齐。

只有形容词、少量口头禅或单一情绪时，继续扩大调研范围；若全网合理可访问资料确实只有这些内容，记录 `已穷尽` 和覆盖缺口，仍以全部可得资料生成正式版。

去名盲测和相似角色区分不得由生成同一批回答的上下文自评。使用独立的人、独立 Agent 或不接收生成过程与角色答案的隔离上下文，并保存样本数、失败样本和原始记录位置。

## 5. 隐私、版权与真实性

- 不将用户资料、生成的人格或来源副本自动上传。
- 不在普通回复中连续复述长篇原文。
- 默认将资料范围标记为“仅本地”。用户可以在其确认有权使用的前提下，导入、保留和分析完整对白、长篇台词、剧本或字幕；它们用于私有资料层和卡片蒸馏，不因长度被程序拒收。
- 私有化不等于自动获得版权、下载权或再分发权。版权属性本身不是私有收集的过滤条件；用户负责确认资料的取得、使用与保存范围，Agent 只记录该声明，不作法律结论。不得绕过登录、付费或访问控制。
- 公开分发、上传或同步的范围由用户负责确认。Agent 只提醒风险，不因版权类别或长度阻止私有收集，也不擅自删除资料；用户要求清理版时再生成独立副本。
- 精确引文必须保留原文并标明来源；推导和改写不能加引号冒充原话。
- 公开前检查私人姓名、联系方式、凭据、本地绝对路径、用户未确认可公开的文本和可识别的私密对话。
