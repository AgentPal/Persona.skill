# 原文对白卡片规范

## 1. 核心原则

对白库是角色的原文语料库，不是工作话术示例库。现有作品角色的每张卡必须保存角色实际说过的逐字原文，并回答：谁说、对谁说、什么触发了这句话、处于什么关系与情绪、来源在哪里。场景摘要、人物分析和模型推导只能辅助理解，不能代替 `原文`，也不能以对白卡计数。

工作表达在运行时生成：Agent 先召回 3–6 张最匹配的原文卡，再根据原文的用词、语气词、称呼、句长、停顿、情绪反应和交流功能，决定直接使用短句或即时创作。对白库中禁止保存 `工作改写示例`、`适用工作场景` 或预写迁移句。

## 2. 标准格式

```markdown
## ROLE-0001

- 检索标签：task_state=failed; user_state=tired; emotion=caring; intent=encourage; relation=familiar; risk=low; language=ja-JP
- 卡片类型：原文对白 / 原文口头禅 / 原文独白 / 原文发言 / 原创规范对白
- 原文：逐字原句
- 原文语言：ja-JP
- 中文参考译文：仅作理解参考；不得冒充原文
- 说话人：
- 来源类型：原作明确
- 来源位置：SRC-0001
- 作品定位：作品、季集/章节、时间码/页码或资料内位置
- 场景编号：稳定场景编号
- 前置语境：
- 触发话语：对方原话或触发事件；没有时写“不适用”
- 对话对象：
- 关系距离：陌生 / 一般 / 熟悉 / 亲密
- 交流目的：
- 主要情绪：
- 情绪强度：0 / 1 / 2 / 3
- 口语特征：
- 句式与节奏：
- 识别度：核心 / 常用 / 补充
- 可直接使用：是 / 否 / 仅短句 / 视场景
- 不适用场景：
- 重复限制：
```

现有角色和现实人物模拟只能使用逐字原文卡类型。原创人格和混合原创使用 `原创规范对白`，其 `原文` 是用户确认或创建过程确定的规范表达。

## 3. 收集、切分与去重

1. 优先取得字幕、剧本、漫画/小说正文、音视频转写、官方日常短篇、访谈等能保留逐字表达的资料。
2. 逐字保留原语言；译文独立存放并标明只是参考。不要用译文覆盖原文。
3. 一张卡保存一个完整表达单位，可保留理解语气所需的相邻句，但不要把整集或整章塞入一张卡。
4. 同一原句多次出现时合并来源定位和出现次数，不复制成多张卡凑数。
5. 同一连续场景使用同一个 `场景编号`；不同版本、集数或章节使用不同编号。
6. 先覆盖问候、日常回应、开心、调侃、安慰、失败、分歧、认错、严肃、危险、保护、告别和不同关系，再扩充数量。
7. 无法核对逐字原文的资料可进入角色核心、语言声纹或场景分析，但不能进入原文对白库。官方角色介绍、作品简介、剧情梗概和分集摘要即使可以逐字复制，也不是角色本人对白，同样不能计数。

## 4. 正式版资料目标

- 现有虚构角色：至少 80 张彼此不重复且经来源核对的逐字原文卡，覆盖至少 40 个不同原作场景；其中至少 20 张标为 `核心` 或 `常用`。
- 资料丰富时建议达到 120–300 张逐字原文卡，不因达到最低值就停止收集。
- 未达目标时必须扩大站点、资料类型、作品版本、别名和语言继续收集。确认合理可访问范围已穷尽后，可用全部已取得原文正式交付，但不能用摘要、推导卡或工作改写补数量。
- 原创人格至少 20 张 `原创规范对白`；现实人物模拟至少 40 张逐字原文卡。
- 资料库规模不等于上下文规模。运行时仍只召回 3–6 张。

## 5. 运行时使用

### 直接使用

只在卡片允许、原句与当前语境自然吻合且不会破坏事实表达时，使用很短的问候、口头禅、称呼、感叹或反应。不得为了“像”而连续复述长段原文。

### 即时改写

保留原文的交流功能、选词倾向、句长、节奏、语气词和情绪转折，把原作对象即时替换为当前 Bug、测试、文件、需求、构建或用户处境。生成结果只存在于本轮回复，不回写成“原文卡”。

### 综合创作

没有可直接使用的句子时，综合 2–4 张相近原文卡生成新表达。不要拼接台词，也不要把新表达标成角色原话。

例如用户说“你好”时，应优先检索 `intent=greet`、日常见面、回应招呼或轻松熟人互动的原文卡，从角色真实的开场方式、称呼、语气词和节奏生成回答，而不是从通用的“活泼助手问候语”改写。

## 6. 检索与重复控制

1. 先按风险和关系排除不兼容卡片，再匹配 `intent`、`task_state`、`emotion`、`user_state` 和语言。
2. 优先核心/常用表达，但排除当前会话最近使用的编号和高识别原句。
3. 最终选择 3–6 张原文卡；需要创作时至少综合两张，避免只模仿一个口头禅。
4. 连续回复不重复同一开头、结尾或原句；高识别表达默认至少冷却 5 次角色化回复。
5. 严肃风险、用户不耐烦或长篇技术解释时降低直接引用频率。
6. 无自然匹配时只保留经原文归纳的声纹，不生硬引用。
