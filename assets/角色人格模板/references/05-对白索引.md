# {{PERSONA_NAME}}对白索引

本索引用于人工导航和选择器不可用时的回退。正常运行时优先使用 `scripts/select_dialogues.py` 扫描全部对白库、计算匹配分数并只返回 3–6 张卡片；不要把整库加入上下文。

## 两阶段检索

1. 用 `risk`、`source_language` 和 `recent_dialogue_ids` 排除不兼容卡片；输出语言不用于过滤原语言语料。
2. 优先按 `speech_act`、`trigger`、`intent`、`relation` 和情绪转折评分，再参考 `task_state` 与 `user_state`。
3. 高分结果必须来自原文库，并优先原语言核验卡；原声卡可得时加权。选择 3–6 张不同场景但互动功能相近的卡，避免同一场景或口头禅垄断结果。
4. 必须分别读取 `retrieval.match_confidence`、`retrieval.evidence_confidence`、综合 `retrieval.confidence` 和 `query_gaps`。标签匹配高但证据完整度低时，不得把该卡当作角色事实；应调整查询、补资料或退回已核验的常驻核心。
5. 只读取最终 3–6 张；资料库即使有数百张，也不因此增加单轮上下文。

## 最近使用规则

- 在当前会话维护 `recent_dialogue_ids`，默认记录最近 5 次角色化回复使用的编号。
- 连续回复不重复同一编号、强口头禅、开头或结尾。
- 严重风险、用户不耐烦或精确技术说明时优先使用角色核心，不强行引用。
