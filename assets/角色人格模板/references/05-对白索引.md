# {{PERSONA_NAME}}对白索引

本索引用于人工导航和选择器不可用时的回退。正常运行时优先使用 `scripts/select_dialogues.py` 扫描全部对白库、计算匹配分数并只返回 3–6 张卡片；不要把整库加入上下文。

## 两阶段检索

1. 用 `risk`、`language` 和 `recent_dialogue_ids` 排除不适合的卡片。高风险场景只保留明确适用于高风险的资料。
2. 按 `task_state`、`intent`、`user_state`、`emotion`、`relation`、`risk` 和 `language` 加权评分，优先选择任务阶段与交流目的同时匹配的卡片。
3. 高分结果必须来自原文对白库；需要即时创作时尽量选择 2–4 张来自不同场景、但交流功能相近的原文卡，避免全部依赖同一口头禅。
4. 只读取最终 3–6 张；资料库即使有数百张，也不因此增加单轮上下文。

## 最近使用规则

- 在当前会话维护 `recent_dialogue_ids`，默认记录最近 5 次角色化回复使用的编号。
- 连续回复不重复同一编号、强口头禅、开头或结尾。
- 严重风险、用户不耐烦或精确技术说明时优先使用角色核心，不强行引用。

## 搜索示例

```text
python scripts/select_dialogues.py --task-state failed --user-state tired --emotion caring --intent encourage --relation familiar --risk medium --language zh-CN --limit 5 --exclude {{CARD_PREFIX}}-0001
```
