# {{PERSONA_NAME}}对白索引

本索引用于人工导航和选择器不可用时的回退。正常运行时优先使用 `scripts/select_dialogues.py` 扫描全部对白库、计算匹配分数并只返回 3–6 张卡片；不要把整库加入上下文。

## 两阶段检索

1. 用 `risk`、`source_language` 和 `recent_dialogue_ids` 排除不兼容卡片；输出语言不用于过滤原语言语料。
2. 先用工作迁移表把 `task_state`、`user_state` 与 `risk` 转成原作层 `speech_act`、`trigger`、`interaction`、`position`、`relation`、`emotion` 和 `initiative`，再评分。工作标签不得进入原文卡。
3. 高分结果必须来自原始表达库，并优先使用来源核验且媒介语境充分的卡；原声、版式或原始发布页面可得时加权。最多选择 3–6 张来自不同证据单元但互动功能相近的卡，避免同一对话、章节、采访或帖子垄断结果。
4. 必须分别读取 `retrieval.match_confidence`、`retrieval.evidence_confidence`、综合 `retrieval.confidence`、`composition_guidance.generation_readiness` 和 `query_gaps`。前三者只评价召回，不能当作成品角色还原度；生成准备度 low 时不得硬发。只使用同时通过检索条件与逐卡观察映射的 `related_rules`。
5. 只读取最终 3–6 张；资料库即使有数百张，也不因此增加单轮上下文。

## 最近使用规则

- 在当前会话维护 `recent_dialogue_ids`，默认记录最近 5 次角色化回复使用的编号。
- 连续回复不重复同一编号、强口头禅、开头或结尾。
- 严重风险、用户不耐烦或精确技术说明时优先使用角色核心，不强行引用。
