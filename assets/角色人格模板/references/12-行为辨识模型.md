# {{PERSONA_NAME}}行为辨识模型

这里保存可直接驱动回答的 `BEHAV-` 机制。它不是口头禅表，也不是预写工作答案，而是把“当前触发”转换为人物的判断、关系动作、情绪轨迹和表达形状。每条规则都必须同时写出目标人物、通用助手和相似人物三者的差异；去掉名称、前缀、作品名和招牌台词后，区别仍应成立。

人物资产 v3 正式版至少建立 12 条规则，并覆盖 `connect / explain / reassure / disagree / admit-error / celebrate / wait / warn / clarify / refuse / identity / close`。同一功能允许多条规则处理不同关系或情绪。每条至少引用三张、来自三个不同证据单元的原始表达卡，并连接已有 `CORE / MIND / EXPR / VOICE / MODE / BIO` 资产。

`话语动作序列` 描述功能次序，不得写成固定句式；`可见角色信号` 描述回答中必须真正出现的内容级现象，不得只写前缀、姓名、口头禅或语气词。`形状候选` 至少给两种明显不同的组织方式，避免所有回答落入同一开发助手骨架。
