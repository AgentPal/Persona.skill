# Persona.skill

Persona.skill 可以把虚构角色、原创人格或你有权使用的人物资料，整理成一个独立的角色人格 Skill。

角色人格启用后，Agent 会用目标角色的身份、性格和表达方式与你交流；每条 Agent 对话和任务进度都会以 `角色名：` 开头，让你清楚知道人格正在生效。代码、命令、文件、日志、事实和风险判断不会被改写。

本仓库是创建人格的“母 Skill”，不内置任何具体作品的大型对白库。

## 安装

下载这个仓库，把整个文件夹改名为 `persona`，放到对应目录：

- Codex：`~/.codex/skills/persona`
- Claude Code：`~/.claude/skills/persona`

也可以用 Git：

```powershell
# Codex
git clone https://github.com/AgentPal/Persona.skill.git "$HOME\.codex\skills\persona"

# Claude Code
git clone https://github.com/AgentPal/Persona.skill.git "$HOME\.claude\skills\persona"
```

如果 Agent 已经打开，安装后重新开启一次会话。

## 使用

直接告诉 Agent：

```text
使用 Persona.skill，根据我提供的资料创建一个角色人格 Skill。
```

创建开始时，Persona.skill 会先让你选择名称：

```text
请先选择角色人格的名称：
1. 使用原角色名：锦木千束
2. 自定义角色名（请直接告诉我名称）
```

这一步不会跳过。选好名称后，调研进度、问题、校验和完成报告都会以 `角色名：` 开头。创建完成后默认在当前会话启用，并在可以安全确定配置位置时持久绑定当前项目；如果只想生成文件，请明确说“只创建不启用”。

启用后，你会看到类似这样的消息：

```text
锦木千束：好，先让我看看这次要处理哪些文件。
```

你也可以说：

```text
使用 Persona.skill 创建一个原创的温柔搭档人格，先通过对话问我需要的信息。
```

创建完成后，可以继续说：

```text
停用当前人格。
恢复默认表达。
卸载这个人格，但保留 Persona.skill。
```

启用时默认只作用于当前项目，并且同一作用域只启用一个人格。用户资料和生成的人格默认保存在本地；是否调研、安装、启用或公开分享，都由你决定。

## 需要准备什么

创建已有角色时，只提供一个角色名字就可以开始。Persona.skill 默认联网深度调研官方或可核查资料，整理角色设定、不同关系与情绪场景、代表性短句和语言规律，并建立来源索引；只有你明确要求“不要联网”时才只使用本地资料。

你也可以补充人物介绍、代表性场景、对白笔记、字幕、文章、聊天记录或自己写的工作对话示例。资料越能覆盖成功、失败、安慰、分歧和风险等不同场景，角色越稳定。

默认是私有本地使用：你可以在确认自己有权使用的前提下，导入和保留完整对白、长篇台词、剧本或字幕，用来建立更丰富的场景库；程序不会因为文本较长而拒绝它们。运行时仍只会挑选最匹配的 3–6 张卡片，不会把整份资料加入上下文。

私有保存不等于自动获得版权或再分发权，请自行确认资料的取得、使用、保存和发布范围；不要绕过付费或访问限制。Persona.skill 不会按版权类别、文本长度或资料完整度拒收私有资料。公开分享时，你可以按自己的权利范围直接发布，或让 Agent 另行生成清理后的副本。

Persona.skill 只交付正式版。已有虚构角色以 80 张有效卡片、40 张原作场景卡和 20 张有来源的原作短句卡为收集目标；资料丰富时建议整理 120–300 张。第一轮资料不足时会自动扩大站点、资料类型、版本、别名和语言继续调研。只有确认合理可访问的全网资料已经穷尽后，才会使用当前收集到的全部内容正式交付，并写明扩大范围和资料缺口。资料库可以很大，但运行时每次只筛选最匹配的 3–6 张。

使用现实人物资料时，请确认你有权使用，并把生成结果明确当作角色模拟，不要用于冒充本人。
